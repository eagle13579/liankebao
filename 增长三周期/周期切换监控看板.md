# 周期切换监控看板

> 来源：增长三周期·M10(周期切换信号识别)
> 用途：持续监控链客宝所处的增长阶段，在灰色地带做数据驱动判断

---

## 一、破局期→增长期切换信号

| 信号指标 | 破局期特征 | 增长期特征 | 当前值 | 状态 |
|:---------|:-----------|:-----------|:------|:----:|
| 用户主动推荐 | 极少 | NPS>30，推荐开始发生 | | 🔴/🟡/🟢 |
| 获客来源认知 | 模糊 | 清楚TOP3渠道+原因 | | 🔴/🟡/🟢 |
| 留存曲线 | 持续下降 | 开始变平，月流失<10% | | 🔴/🟡/🟢 |
| 核心用户数 | <100 | >200连续3周登录 | | 🔴/🟡/🟢 |
| 付费用户比例 | <5% | >15% | | 🔴/🟡/🟢 |

**切换判断**：≥3个🟢 → 可以进入增长期
**否则**：继续留在破局期，聚焦PMF验证

---

## 二、增长期→升级期切换信号

| 信号指标 | 增长期特征 | 升级期特征 | 当前值 | 状态 |
|:---------|:-----------|:-----------|:------|:----:|
| 核心增速 | >20%月增长 | 连续2-3季度个位数 | | 🔴/🟡/🟢 |
| CAC趋势 | 稳定或下降 | 持续上升 | | 🔴/🟡/🟢 |
| 团队状态 | 高效执行 | 开始内卷，动作变形 | | 🔴/🟡/🟢 |
| 不用你的人群 | 少 | 在扩大（因买不到而忍受） | | 🔴/🟡/🟢 |
| 新替代品出现 | 无 | 跨界竞品或技术变化 | | 🔴/🟡/🟢 |

**切换判断**：≥3个🔴 → 需要考虑进入升级期
**否则**：继续聚焦增长期飞轮优化

---

## 三、监控频率

| 阶段 | 监控频率 | 负责人 |
|:-----|:---------|:-------|
| 破局期 | 每周 | 乘黄（产品经理） |
| 增长期 | 每月 | 乘黄+计然（产品+战略） |
| 升级期 | 每季度 | 白泽+计然（CEO+战略） |

---

## 四、监控数据源

| 指标 | 数据来源 | 是否需要新增埋点 |
|:-----|:---------|:----------------|
| 留存率 | 用户行为数据库 | ✅ 需确认 |
| NPS | 用户调研 | ❌ 已有 |
| CAC | 财务/市场数据 | ❌ 已有 |
| 增速 | 业务数据库 | ✅ 需确认 |
| 非用户人群 | 市场调研 | ✅ 需新增 |
