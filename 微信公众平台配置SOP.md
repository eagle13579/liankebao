# 第一步：登录微信公众平台

**操作**:
1. 打开浏览器访问: **https://mp.weixin.qq.com**
2. 用你的微信扫码登录（必须是小程序的管理员/开发者账号）
3. 登录后，在顶部导航栏点击 **「小程序」**（不是公众号！）

**确认**:
- 页面左上角显示你的小程序名称
- 左侧菜单能看到「功能」「开发」「版本管理」等栏目

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第二步：配置服务器域名（必做，否则API请求失败）

> **路径**: 左侧菜单 → **开发** → **开发管理** → 顶部Tab点 **「服务器域名」**

**操作**:
1. 在 **「request合法域名」** 输入框中输入：
   ```
   https://www.go-aiport.com
   ```
2. 点击右侧 **「添加」** 按钮

**确认**:
- 列表中显示 `www.go-aiport.com` ✅
- request合法域名必须写 `https://` 前缀

**为什么**: 小程序的 wx.request() 只能请求白名单内的域名。不配的话，所有API调用都会报 `https://www.go-aiport.com 不在以下 request 合法域名列表中`。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第三步：配置订单中心Path

> **路径**: 左侧菜单 → **功能** → 找到 **「订单中心」**

**操作**:
1. 在订单中心页面点击 **「设置」** 或 **「配置」** 按钮
2. 在弹出的配置框中：
   - **Path 填写**: `pages/orders/index`
   - **测试链接**: 如果有点击测试，应该跳转到订单列表页
3. 点击 **「保存」**

**确认**:
- 页面显示「订单中心已配置」✅
- 路径显示为 `pages/orders/index`

**为什么**: 微信要求小程序声明订单相关页面的路径，用于用户从「订单中心」入口直接跳转。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第四步：更新用户隐私保护指引（审核必过项）

> **路径**: 左侧菜单 → **功能** → 找到 **「用户隐私保护指引」**

**操作**:
1. 点击 **「更新」** 按钮
2. 在弹出的表单中，勾选以下收集的信息：
   ```
   ☑ 微信昵称、头像（用户资料）
   ☑ 手机号
   ☑ 收货地址（产品配送需要）
   ☑ 订单信息
   ```
3. 在 **「用途说明」** 文本框中填写：
   ```
   用于用户注册登录、商品配送、订单管理和售后服务
   ```
4. 点击 **「提交」**

**确认**:
- 页面显示「隐私保护指引已更新」✅

**为什么**: 2023年9月起，微信要求所有小程序必须配置隐私指引，否则提交审核会被驳回。你可以在 `D:\链客宝\submission_checklist.md` 查看完整清单。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第五步：开通微信支付

> **路径**: 左侧菜单 → **功能** → 找到 **「微信支付」**

**操作**:
1. 点击 **「开通」** 按钮
2. 跳转到微信支付商户平台注册页面
3. 填写商户资料:
   - 营业执照信息
   - 法人身份信息
   - 对公账户信息
   - 经营类目选择: **「电商平台」** 或 **「小程序商城」**
4. 提交后等待审核（通常1-3个工作日）

**开通后需要做的事**:
5. 拿到商户号（MCH_ID）后，在链客宝服务器上设置环境变量：
   ```
   登录服务器: ssh opc@47.100.160.250 -p 2222
   编辑: sudo vi /etc/systemd/system/chainke.service
   在 [Service] 段添加:
   Environment="WEIXIN_APPID=你的APPID"
   Environment="MCH_ID=你的商户号"
   Environment="API_KEY=你的API密钥"
   Environment="NOTIFY_URL=https://www.go-aiport.com/lkapi/pay-notify"
   重启: sudo systemctl daemon-reload && sudo systemctl restart chainke
   ```

**完整配置参考**: `D:\链客宝\.env.example`

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第六步：上传小程序代码（用开发者工具）

> **打开**: `D:\链客宝\打开微信小程序.bat`（双击运行）
> 或手动: 微信开发者工具 → 导入 `D:\链客宝\liankebao-miniapp`

**操作**:
1. 打开微信开发者工具
2. 点击 **「导入项目」**
3. 项目目录选择: `D:\链客宝\liankebao-miniapp`
4. APPID: 输入你的小程序APPID
5. 点击 **「导入」**

6. 在开发者工具中点击顶部工具栏的 **「上传」** 按钮
7. 版本号填写: `1.0.0`
8. **版本描述** 粘贴以下内容（已预存在 `D:\链客宝\version_description.txt`）：

```
链客宝企业家供需匹配平台MVP
功能：产品发布浏览、下单购买、推广分润、微信登录
技术栈：FastAPI + React + 微信小程序
后端API已配置SSL和WAF安全防护
```

9. 点击 **「上传」**
10. 上传成功后，在 **「版本管理」** 中能看到开发版本

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 第七步：提交审核

> **路径**: 微信公众平台 → 左侧菜单 → **「版本管理」**

**操作**:
1. 在「开发版本」列表中找到刚上传的版本
2. 点击右侧的 **「提交审核」** 按钮
3. 在弹出的对话框中:
   - **功能页面**: 勾选首页和核心功能页面
   - **测试账号**: 使用 `D:\链客宝\test_account_info.md` 中的4个角色
   - **隐私说明**: 确认已更新（第四步）
4. 点击 **「提交审核」**

**审核周期**: 通常 1-7 天（最快几小时）
**被驳回怎么办**: 看驳回原因 → 修改 → 重新提交

# 第八步：审核通过后发布

**操作**:
1. 微信公众平台 → 版本管理 → 审核通过版本
2. 点击 **「发布」**
3. 确认发布范围（通常选「全量发布」）
4. 完成 ✅ 用户可以通过微信搜索小程序名称访问

# 常见问题

| 问题 | 原因 | 解决 |
|:-----|:------|:------|
| 请求报 `http://` 不在合法域名列表 | 没配request域名 | 去第二步配 `https://www.go-aiport.com` |
| 登录报 `code无效` | appid/secret没配对 | 检查服务器环境变量 |
| 支付弹不了 | 没开通支付/参数未配置 | 完成第五步 |
| 审核被拒「隐私指引未更新」 | 第四步没做 | 去更新隐私指引 |
| 审核被拒「类目不符」 | 选择的类目不对 | 选「电商平台」或「工具-信息查询」 |
| 页面空白 | 前端构建有问题 | 重新 `npm run build` 后上传 |
| API返回401 | token过期 | 重新登录获取新token |
| API返回429 | 速率限制触发 | 等60秒再试（安全防护正常） |

# 附件：链客宝服务器环境变量清单

SSH到服务器后，编辑chainke.service配置:

```bash
ssh opc@47.100.160.250 -p 2222
sudo vi /etc/systemd/system/chainke.service
```

在 `[Service]` 段添加:

```
Environment="SECRET_KEY=你的JWT密钥（随机字符串）"
Environment="ACCESS_TOKEN_EXPIRE_MINUTES=30"
Environment="REFRESH_TOKEN_EXPIRE_DAYS=7"
Environment="CORS_ORIGINS=https://www.go-aiport.com,https://liankebao.top"
Environment="WEIXIN_APPID=你的小程序APPID"
Environment="MCH_ID=你的微信商户号"
Environment="API_KEY=你的微信支付API密钥"
Environment="NOTIFY_URL=https://www.go-aiport.com/lkapi/pay-notify"
Environment="SENTRY_DSN=（可选，监控用）"
```

保存后:
```bash
sudo systemctl daemon-reload && sudo systemctl restart chainke
journalctl -u chainke -n 20 --no-pager
```
