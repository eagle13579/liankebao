## 二、缺失的关键集成 (TOP-5)

### #1 微信支付集成  —— 最高优先级

**现状**: gateway 已路由 `/api/v1/membership/` -> :8000 (fastapi_payment), 但:
- `backend/app/routers/payment.py` **不存在** (仅在 replace_appid.py 中被引用, 但文件缺失)
- `backend/app/routers/orders.py` **不存在**
- 会员额度在 `membership.py` 中是内存存储 (`_CREDITS_STORE: dict[int, int]`), 无真实支付
- 有完整的设计文档 (`WECHAT_PAYMENT_PLAN.md`, `WECHAT_PAY_SETUP_CHECKLIST.md`) 但未落地

**预估ROI**: ★★★★★ (极高)
- 会员付费是B2B匹配平台的核心盈利模式
- 当前额度系统纯内存实现, 无法真正收款
- 微信支付覆盖中国90%+移动支付用户

**工作量**: 3-5天
- SDK集成: 1天 (微信支付Python SDK / 原生API封装)
- 订单系统: 1天
- 回调通知: 1天
- 退单/对账: 1-2天

### #2 短信/邮件通知集成  —— 高优先级

**现状**: 完全缺失
- 无短信服务 (无阿里云短信/腾讯云短信/ twilio)
- 无邮件发送服务 (无 sendgrid / smtplib 封装)
- onboarding 表单收集了 phone/email, 但无法发送通知
- 飞书告警仅限运维场景, 非面向用户

**预估ROI**: ★★★★☆ (高)
- 匹配成功后需要通知用户 (核心场景)
- 验证码登录 (当前无短信验证码)
- 营销触达 (活动通知/会员提醒)

**工作量**:
- 短信 (阿里云SDK): 1天 ← 1天内可完成
- 邮件 (smtplib + 模板): 0.5天 ← 1天内可完成

### #3 企业微信/钉钉/飞书IM集成  —— 高优先级

**现状**: 完全缺失
- 脚本层有飞书 Webhook 告警 (运维通知), 但无 Bot/消息推送给用户
- 无企业微信应用集成
- 无钉钉机器人/应用
- 链客宝是B2B匹配平台, 企业IM是触达企业用户的天然渠道

**预估ROI**: ★★★★ (高)
- B2B用户活跃在企微/钉钉
- 匹配结果推送、商机提醒可直达企业用户
- 企业微信小程序互通能力强

**工作量**:
- 飞书 Bot (Webhook): 0.5天 ← 1天内可完成
- 钉钉机器人: 0.5天 ← 1天内可完成
- 企业微信应用: 需要商务对接 (企业微信审核, 应用市场上架)

### #4 数据分析/用户行为分析  —— 中高优先级

**现状**: 完全缺失
- 无埋点系统 (无 GrowingIO / SensorsData / Umami / GA4 / Mixpanel)
- 无用户行为分析基础设施
- 有 data_pipeline 配置 (features/data_pipeline/) 但仅是内部数据流水线
- retention_insights 和 unit_economics 是静态模拟数据, 无真实数据接入

**预估ROI**: ★★★★ (高)
- 无法衡量用户留存、转化漏斗
- 无法做数据驱动决策
- 产品迭代失去方向

**工作量**:
- 自建 Umami: 1天 ← 1天内可完成
- 集成 GrowingIO SDK: 1天 ← 1天内可完成
- 集成 SensorsData: 2天

### #5 文件存储/CDN  —— 中优先级

**现状**: 完全缺失
- 无对象存储 (无阿里云OSS / 腾讯云COS / 七牛云 / MinIO / AWS S3)
- business_card 和 brochure 的图片/文件存储全在本地文件系统
- 无 CDN 加速
- 无图片处理/缩略图能力

**预估ROI**: ★★★☆ (中高)
- 名片封面图、企业资料需要可靠存储
- 本地存储无法水平扩展
- 无CDN导致加载慢

**工作量**:

- 七牛云SDK: 1天 ← 1天内可完成
