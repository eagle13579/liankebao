# 链客宝安全策略

> 版本: 1.0.0 · 更新日期: 2026-06-24

---

## 概述

本文档描述链客宝（ChainKe）项目的安全策略、实践准则和应急响应流程。所有贡献者和维护者应遵守本文档中的安全要求。

**安全当前评分:** 3/10 → 目标: 10/10

---

## 1. 密钥管理

### 1.1 禁止提交

以下类型的文件**严禁**提交到版本控制系统：

- `.env` 文件（包含 `SECRET_KEY`、`WECHAT_APP_SECRET` 等）
- 任何 `.pem`、`.p12`、`.pfx`、`.key` 私钥文件
- API 密钥、令牌硬编码在源代码中
- 数据库文件（`*.db`、`*.sqlite`、`*.jsonl`）

### 1.2 安全做法

- 使用 `.env.example` 提供环境变量模板，**不包含真实密钥**
- 所有密钥通过环境变量或安全的密钥管理服务注入
- 定期轮换密钥（建议每 90 天）

---

## 2. 代码安全

### 2.1 安全检查工具

| 工具 | 用途 | 运行方式 |
|------|------|----------|
| Bandit | Python 安全漏洞扫描 | `pre-commit` 自动运行 |
| pre-commit | 提交前安全钩子 | `pre-commit install` |
| pip-audit | Python 依赖漏洞扫描 | `pip-audit -r requirements.txt` |
| 安全检查脚本 | 10 项综合检查 | `./scripts/security-check.sh` |

### 2.2 开发流程

1. **提交前**: `pre-commit` 自动执行 bandit、私钥检测、大文件检查
2. **CI 中**: 运行 `./scripts/security-check.sh` 作为门禁
3. **定期**: 每周执行一次全量依赖漏洞扫描

---

## 3. 依赖安全

- 所有依赖必须固定版本号（如 `fastapi>=0.104.0,<1.0.0`）
- 禁止直接引入已知有 CVE 漏洞的依赖版本
- 使用 `pip-audit` 或 `safety` 进行依赖审查
- 依赖更新需经过 Code Review

---

## 4. 运行时安全

### 4.1 CORS 配置

CORS 配置应仅允许受信任的域名。当前配置允许的来源：

- `http://localhost:3099`
- `http://localhost:3000`
- `https://liankebao.top`
- `https://www.liankebao.top`

**禁止**在生产环境使用 `allow_origins=["*"]`。

### 4.2 文件权限

- `.env` 文件权限必须设置为 `600`（仅所有者可读写）
- 私钥文件权限必须设置为 `600`
- 日志文件不应包含敏感信息（密码、令牌等）

### 4.3 审计日志

所有管理操作和敏感数据访问应记录审计日志。审计日志保留至少 90 天。

## 5. 漏洞报告

### 5.1 报告渠道

如果发现安全漏洞，**请勿**公开提交 Issue。请通过以下方式报告：

- **Email**: security@liankebao.top（国内镜像）
- **PGP 密钥**: [链接待添加]
- **紧急情况**: 直接联系项目维护者

### 5.2 响应承诺

| 阶段 | 时间 |
|------|------|
| 确认收到 | 24 小时内 |
| 初步评估 | 72 小时内 |
| 修复发布 | Critical: 7 天 · High: 14 天 · Medium: 30 天 |

## 6. 数据保护

- 用户密码必须使用 bcrypt（或 argon2）哈希存储
- 数据库文件不得提交到 Git
- `data/` 目录下的租户数据受 `.gitignore` 保护
- 个人身份信息（PII）需遵循中国《个人信息保护法》要求

---

## 7. 基础设施安全

- Docker 容器以非 root 用户运行
- Nginx 反向代理配置 HTTPS 强制跳转
- 使用 `.dockerignore` 排除敏感文件
- 生产环境启用 WAF（Web 应用防火墙）

---

## 8. 合规性

链客宝项目遵循以下安全合规要求：

- **《网络安全法》** — 等级保护 2.0 基本要求
- **《个人信息保护法》** — 个人信息处理规则
- **OWASP Top 10** — Web 应用安全最佳实践
- **ISO 27001** — 信息安全管理体系框架
