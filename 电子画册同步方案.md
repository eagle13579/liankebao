# 需求2+3 架构方案

## 现状
- 链客宝后端(:8001) 产品发布API → 写入产品表
- AI数字名片(:8003) 有 brochure API（/api/brochures CRUD）
- 网页版有产品池（/product-pool）
- AI数字名片有独立H5页面（/brochure_editor.html）

## 需求2方案：发布产品自动生成电子画册

后端产品发布API中增加：创建产品成功后，调用AI数字名片API同步创建brochure

```
用户发布产品 → POST /api/products
  → 链客宝后端创建产品记录
  → 调用 :8003/api/brochures (POST) 创建电子画册
  → 返回产品+画册ID
产品池页面 → 展示产品 + 「查看电子画册」按钮
```

## 需求3方案：小程序画册在网页版呈现

AI数字名片H5（/brochure_editor.html 和 /api/brochure/{id}/brochure）
已经可以在浏览器中查看。网页版产品池中加一个「电子画册」入口链接即可。

```
网页版产品池 → 每个产品卡片增加「电子画册」按钮
  → 跳转 https://liankebao.top/api/brochure/{id}/brochure
  → 或跳转 /brochure_editor.html?id={id}
```
