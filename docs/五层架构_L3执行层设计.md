## 1. 概述

### 1.1 定位

L3 执行层位于 L2 调度层之下、L4 值守层之上，是**AI 能力的实际执行单元**。接收 L2 调度层的匹配任务，并行派出 3 路子代理执行：

1. **合规诊断 Agent** — 检查需求方资质与合规风险
2. **匹配评分 Agent** — 行业×规模×地域×阶段 四维评分
3. **文档生成 Agent** — 生成推荐报告与合作意向书

### 1.2 当前问题

| 问题 | 现状 | 影响 |
|------|------|------|
| 无 AI 能力嵌入 | 匹配靠人工，无自动评分 | 效率低、主观性强 |
| 无合规诊断 | 直接对接，无风险预检 | 潜在合规风险 |
| 无文档生成 | 推荐结果需要手动整理 | 运营成本高 |

### 1.3 设计目标

1. **AI 匹配评分引擎** — 四维评分（行业×规模×地域×阶段），可解释性好
2. **合规预检 Agent** — 对接獬豸技能，自动化筛查
3. **推荐报告自动生成** — 匹配成功后自动生成推荐简报
4. **可扩展的 Agent 委派框架** — 方便后续添加新 Agent

## 3. AI 匹配评分引擎（核心）

### 3.1 四维评分体系

评分采用 **加权四维模型**，每个维度 0~100 分，最终加权总分归一化到 0.0~1.0：

| 维度 | 权重 | 说明 | 数据来源 |
|:----:|:----:|------|---------|
| 行业匹配 | 35% | 需求品类与产品类目的对齐度 | BusinessNeed.category × Product.category |
| 规模匹配 | 25% | 预算范围与产品价格区间的重叠度 | BusinessNeed.budget × Product.price |
| 地域匹配 | 20% | 需求地区与产品供应区域的吻合度 | BusinessNeed.region × Product.region（新增）|
| 阶段匹配 | 20% | 需求阶段与产品成熟度的适配度 | BusinessNeed.tags / urgency × Product.status |

#### 3.2.2 规模匹配（25 分）

```python
def score_budget_match(need_budget: str, product_price: float) -> float:
    """
    规模/预算匹配评分 (0~25 分)
    
    策略：
    - 将"10万-50万"格式解析为 [low, high] 区间
    - 计算需求预算区间 [L₁, H₁] 与产品价格 [P] 的重叠度
    - 重叠度公式: (min(H₁, P) - max(L₁, P)) / (H₁ - L₁)
    - 预算为空 = 10 分（中性）
    - 价格为空 = 15 分（中性偏高）
    
    区间归一化：所有金额统一转换为"万元"单位
    """
    return budget_score * 0.25  # 归一化到 25 分
```

#### 3.2.3 地域匹配（20 分）

```python
def score_region_match(need_region: str, product_region: str) -> float:
    """
    地域匹配评分 (0~20 分)
    
    策略：
    - 精确匹配（如"华东"="华东"）→ 20 分
    - 大区包含（如"华东"⊃"上海"）→ 15 分
    - 相邻大区 → 10 分
    - 全国 vs 任何 → 15 分
    - 不匹配 → 0 分
    
    地域层级树（由粗到精）:
    全国 > 华北/东北/华东/华南/华中/西南/西北 > 省级 > 市级
    """
    region_hierarchy = {
        "华北": ["北京","天津","河北","山西","内蒙古"],
        "东北": ["辽宁","吉林","黑龙江"],
        "华东": ["上海","江苏","浙江","安徽","福建","江西","山东"],
        "华南": ["广东","广西","海南"],
        "华中": ["河南","湖北","湖南"],
        "西南": ["重庆","四川","贵州","云南","西藏"],
        "西北": ["陕西","甘肃","青海","宁夏","新疆"],
    }
    # 若需求方填"全国"，所有地区 15 分
    return region_score * 0.20  # 归一化到 20 分
```

#### 3.2.5 总分计算

```python
def calculate_match_score(need: BusinessNeed, product: Product) -> MatchResult:
    """
    综合四维评分，返回可解释的 MatchResult
    
    权重合计: 0.35 + 0.25 + 0.20 + 0.20 = 1.0
    
    返回:
    - score: 0.0~1.0 归一化分数
    - reasons: 各维度的中文评分理由
    - details: 各维度原始分数（用于调试和展示）
    """
    industry = score_industry_match(need.category, product.category)
    budget = score_budget_match(need.budget, product.price)
    region = score_region_match(need.region, product.region)  # Product 需新增 region 字段
    stage = score_stage_match(need.tags, need.urgency, product.status)
    
    total = (industry + budget + region + stage) / 100.0
    reasons = []
    if industry > 0:
        reasons.append(f"行业匹配度 {industry:.0f}/35")
    # ...
    return MatchResult(
        product_id=product.id,
        score=round(total, 4),
        reasons=reasons,
        details={"industry": industry, "budget": budget, "region": region, "stage": stage}
    )
```

## 4. 合规预检 Agent

### 4.1 设计概要

合规预检 Agent 在匹配流程开始前执行，检查需求方的合规风险。对接獬豸技能（xiezhi_agent），若不可达则降级为本地规则检查。

### 4.2 检查维度

| 维度 | 检查项 | 数据源 | 权重 |
|:----:|--------|--------|:----:|
| 企业资质 | 营业执照是否在有效期内 | Enterprise 表 | fatal |
| 行业准入 | 需求品类是否需要特殊许可 | 本地规则库 | high |
| 黑名单 | 企业/个人是否在平台黑名单 | Blacklist 表 | fatal |
| 信用评分 | 用户历史行为有无违约记录 | UserEvent + Order | medium |
| 敏感内容 | 需求描述是否含违规词 | 敏感词库 | high |

### 4.3 结果等级

| 等级 | 说明 | 匹配动作 |
|------|------|---------|
| pass | 合规通过 | 正常匹配 |
| warn | 有轻微风险提示 | 匹配 + 标注 warning |
| block | 高风险，禁止匹配 | 拦截，返回具体原因 |

### 4.4 獬豸技能对接

```python
class ComplianceAgent:
    """合规预检 Agent"""

    def __init__(self):
        self.xiezhi_endpoint = os.getenv("XIEZHI_ENDPOINT", "http://localhost:5051")
        self.fallback_rules = self._load_fallback_rules()

    async def check(self, need: BusinessNeed, user: User) -> ComplianceResult:
        """
        合规预检入口
        
        优先调用獬豸技能 API，若 3 秒内无响应则降级为本地规则检查。
        """
        try:
            result = await self._call_xiezhi(need, user)
            return result
        except (TimeoutError, ConnectionError):
            logger.warning("獬豸技能不可达，使用本地规则降级")
            return self._fallback_check(need, user)

    async def _call_xiezhi(self, need, user) -> ComplianceResult:
        """调用獬豸技能做 AI 合规判断"""
        async with httpx.AsyncClient(timeout=3.0) as client:
            resp = await client.post(f"{self.xiezhi_endpoint}/api/v1/compliance/check", json={
                "enterprise_name": user.company,
                "industry": need.category,
                "description": need.description,
                "contact_name": need.contact_name,
                "contact_phone": need.contact_phone,
            })
            data = resp.json()
            return ComplianceResult(
                status=data["status"],  # pass / warn / block
                risk_level=data.get("risk_level", "low"),
                warnings=data.get("warnings", []),
                details=data.get("details", {}),
                source="xiezhi",
            )

    def _fallback_check(self, need, user) -> ComplianceResult:
        """本地规则降级检查"""
        warnings = []
        # 1. 黑名单检查
        if self._is_blacklisted(user):
            return ComplianceResult(status="block", ...)
        # 2. 敏感词检查
        sensitive = self._check_sensitive_words(need.description)
        if sensitive:
            warnings.append(f"需求描述包含敏感词: {sensitive}")
        # 3. 空字段提醒
        if not user.company:
            warnings.append("需求方未填写企业信息，资质待确认")
        status = "warn" if warnings else "pass"
        return ComplianceResult(
            status=status,
            risk_level="medium" if status == "warn" else "low",
            warnings=warnings,
            source="fallback",
        )
```

### 4.5 合规结果在匹配中的使用

```python
def apply_compliance_to_matches(matches: list, compliance: ComplianceResult) -> list:
    """
    合规结果影响匹配：
    - block → 所有匹配项标记为 blocked，不下发给需求方
    - warn → 匹配项保留，但每个 result 增加 compliance_warning
    - pass → 正常返回
    """
    if compliance.status == "block":
        return []
    for m in matches:
        m.compliance_status = compliance.status
        if compliance.warnings:
            m.compliance_warnings = compliance.warnings
    return matches
```

## 6. Agent 委派调度器（TaskDispatcher）

### 6.1 核心逻辑

```python
class TaskDispatcher:
    """
    L3 执行层 Agent 委派调度器
    
    1. 接收 L2 的匹配任务
    2. 并行派出 3 路子代理
    3. 聚合结果
    4. 回写 L2 回调接口
    """

    def __init__(self, db: Session):
        self.db = db
        self.compliance_agent = ComplianceAgent()
        self.score_engine = MatchScoreEngine()
        self.report_gen = ReportGenerator()

    async def execute(self, task: MatchTask) -> MatchTaskResult:
        """
        执行匹配任务
        
        Step 1: 并行派出合规诊断 + 匹配评分
        Step 2: 等待两路结果
        Step 3: 合规结果过滤匹配列表
        Step 4: 文档生成（异步，不阻塞主流程）
        Step 5: 聚合结果，返回给 L2 回调
        """
        need = self.db.query(BusinessNeed).get(task.need_id)
        user = self.db.query(User).get(need.user_id)
        products = self.db.query(Product).filter(Product.status == "approved").all()

        # Step 1: 并行执行
        compliance_task = asyncio.create_task(
            self.compliance_agent.check(need, user)
        )
        score_task = asyncio.create_task(
            self.score_engine.match_all(need, products)
        )

        compliance, raw_matches = await asyncio.gather(compliance_task, score_task)

        # Step 2: 合规过滤
        final_matches = apply_compliance_to_matches(raw_matches, compliance)

        # Step 3: 取 Top-K
        top_matches = sorted(final_matches, key=lambda m: m.score, reverse=True)[:5]

        # Step 4: 文档生成（后台异步）
        asyncio.create_task(
            self.report_gen.generate_recommendation_report(need, top_matches, compliance)
        )

        # Step 5: 返回结果
        return MatchTaskResult(
            queue_id=task.queue_id,
            status="completed",
            matches=[m.to_dict() for m in top_matches],
            compliance_summary={
                "status": compliance.status,
                "risk_level": compliance.risk_level,
                "warnings": compliance.warnings,
            },
        )
```

## 8. 错误处理与降级策略

| 场景 | 策略 | 影响 |
|------|------|------|
| 獬豸技能不可达 | 降级到本地规则检查 | 合规检查精度降低，不阻塞匹配 |
| 獬豸技能超时（>3s） | 截断，使用已有结果 | 同上 |
| 评分引擎异常 | 降级到 matching_engine v2 | 评分维度变少，但可用 |
| 文档生成失败 | 记录错误，不阻塞主流程 | 报告暂缺，后台可重试 |
| 全部 Agent 异常 | 返回 error 给 L2，L2 标为 failed | 需求标记异常，运营介入 |
