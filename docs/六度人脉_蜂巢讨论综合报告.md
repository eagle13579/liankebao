| 要点 | 内容 |

### 🐉 烛龙（CTO·技术视角）

| 要点 | 内容 |
|:-----|:------|
| **数据模型** | `user_relations`邻接表 + `relation_events` + `six_degree_path_cache` + `referral_links` — SQLite兼容 |
| **算法** | 双向BFS `O(b^(d/2))` — b=50分支因子时，单向156亿→双向25万节点 |
| **信任衰减** | `T(i) = decay^i`，decay=0.6，1度1.0 → 6度0.047 |
| **API设计** | 12个REST端点 + WebSocket实时通知 |
| **性能基线** | 1万用户10-50ms，10万用户50-200ms，100万200ms-1s |
| **缓存策略** | 三级：内存LRU(1万条)→DB持久缓存→Redis(未来) |
| **架构演进** | 阶段1 SQLite(10万) → 阶段2 PostgreSQL(100万) → 阶段3 Neo4j(1000万) |
| **代码就绪** | 4个代码文件已创建：models/six_degrees.py(数据模型)、services/six_degrees.py(BFS算法+信任评分)、routers/six_degrees.py(12API)、routers/six_degrees_ws.py(WebSocket)、migrate.py(迁移脚本) |

## 四、最终推荐方案

### 产品策略（综合乘黄+狴犴意见后修正）

**核心定位**：~~六度人脉~~ → **「信任链」（Trust Chain）**
> 不做「认识所有人」，做「你认识的人帮你信任陌生人」
> 一句话：在链客宝，你找到的供应商不只是算法匹配的，还是你信任的人推荐的。

**默认交互**：
- 搜索深度限制为**3度**（朋友的朋友的朋友）
- 路径展示匿名化：「目标用户张三，你与TA之间隔了2人」
- 引荐需中间人双向确认（发起引荐申请→中间人同意→对接）

**功能模块（MVP，Phase 0-1）**：

| 优先级 | 功能 | 成本 | 开发周期 |
|:------:|:-----|:----:|:--------:|
| P0 | 共同好友检测（名片交换时提示「你们有X个共同好友」） | ¥0（复用现有数据） | 1周 |
| P0 | 邀请链接生成 + 关系边建立（名片交换/邀请注册自动建边） | ¥2万 | 1周 |
| P1 | 人脉路径查询（输入目标→展示3度内路径长度） | ¥5万 | 2周 |
| P1 | 信任链推荐（AI匹配结果加权显示「有共同联系人」） | ¥3万 | 1周 |
| P2 | 引荐工作台（发起引荐请求→中间人确认→对接） | ¥5万 | 2周 |
| P3 | 多级分润（引荐成交后推荐人分润） | ¥3万 | 1周 |

### 技术架构（综合烛龙+狴犴后修正）

**否决**Neo4j规划（现阶段不需要）。
**采纳**SQLite邻接表+双向BFS（烛龙的现有代码正确）。
**修正**缓存策略：加1层无状态缓存（降低复杂度），去掉Redis规划。

```
MVP技术栈：
  数据库: SQLite（已有）+ user_relations邻接表
  算法: 双向BFS（烛龙已实现）
  缓存: 无（<1万用户时不需要）→ 内存LRU（1-10万用户时加）
  API: 6个端点（非12个），去掉WebSocket（改为长轮询）
  前端: 在现有Vue3 SPA中加1个「信任链」页面
  AI联动: matching_engine中加「共同联系人加权」
```

### 架构流程

```
用户A查看供应商B
    │
    ├─ matching_engine 计算出匹配度 (80%)
    │
    └─ six_degrees 计算出信任链
    │   └─ 结果: 「B与你有1个共同好友—张三」
    │   └─ 信任加权: 匹配度80% × 信任系数1.3 = 最终推荐104%
    │
    └─ 展示给A: 「推荐: B公司 ｜ 匹配度80% ｜ 通过张三可对接」
```

### 冷启动策略（综合计然+狴犴后修正）

**否决**C端社交裂变（狴犴正确——B2B不适合拼多多模式）。
**采纳**双钩子冷启动（乘黄方案）+ 行业码（计然补充）。

```
Phase 0 (0→50关系边): 
  - AI数字名片交换自动建边（现有功能，¥0成本）
  - CEO向海容手动邀请20位核心伙伴
  - 种子数据中创建20组「关系示例」展示功能效果

Phase 1 (50→500关系边):
  - 推广员体系激活（现有4个产品的推广员，约10人）
  - 「行业码」模式：给XX商会/协会专属邀请链接
  - 首次对接券激励：前100名完成引荐的用户奖¥50

Phase 2 (500→5000关系边):
  - 信任链推荐上线（AI匹配+信任加权）
  - 成功引荐案例沉淀为品牌素材
  - 网络效应开始显现
```
