# 链客宝 · 主体变更 + 微信小程序配置 手把手SOP

> **场景**: 从「显臻教育」主体变更为「容蓝实业」主体，运营链客宝小程序
> **当前AppID**: `wxb4f6d89904200fd2`（显臻教育）
> **服务器**: 47.116.116.87 | **域名**: liankebao.top
> **预估耗时**: 主体变更3-7天 + 配置30分钟

---

## 第一部分：主体变更（必读）

> ⚠️ 微信小程序**不支持直接修改主体**。有两种方案可选：

### 方案A：小程序迁移（推荐，保留数据）

微信官方支持将小程序从一个主体迁移到另一个主体，**保留AppID、用户数据、配置**。

**前提条件**：
- 原主体（显臻教育）正常经营，营业执照未注销
- 新主体（容蓝实业）已注册并完成微信认证
- 双方管理员微信可扫码确认

**操作步骤**：

**第1步：登录微信公众平台**

打开 https://mp.weixin.qq.com，用显臻教育的管理员微信扫码登录。

**第2步：发起迁移申请**

左侧菜单 → **设置** → **主体信息** → 找到「主体变更」→ 点击「开始迁移」

> 如果找不到入口，直接搜索「小程序迁移」进入功能页面。

**第3步：填写目标主体信息**

在弹出的表单中填写：
```
目标主体: 容蓝实业
统一社会信用代码: [容蓝实业的营业执照号]
法人姓名: [容蓝实业的法人姓名]
管理员手机号: [容蓝实业的管理员手机]
管理员微信号: [容蓝实业的管理员微信号]
```

**第4步：上传资质材料**

上传以下文件（拍照或扫描）：
- ✅ 显臻教育营业执照（原件照片）
- ✅ 容蓝实业营业执照（原件照片）
- ✅ 双方法人身份证正反面
- ✅ 小程序迁移申请函（系统生成，双方盖章）

**第5步：双方管理员扫码确认**

系统会分别向显臻教育和容蓝实业的管理员微信发送确认通知。双方扫码确认后提交审核。

**第6步：支付审核费**

微信收取 **300元** 审核费（与认证费分开）。

**第7步：等待审核**

审核周期：1-7个工作日。
审核通过后，数据自动迁移。**AppID不变**。

---

### 方案B：注销旧小程序 + 重新注册（推荐，更干净）

如果显臻教育的营业执照已注销或无法配合，只能走这条路。

**流程**：
```
注销旧小程序 → 用容蓝实业注册新小程序 → 获取新AppID → 更新代码 → 重新配置
```

**第1步：注销显臻教育的小程序**

1. 登录 https://mp.weixin.qq.com（显臻教育账号）
2. 左侧菜单 → **设置** → **基本设置**
3. 拉到最底部 → **「注销小程序」**
4. 按提示确认（需管理员扫码 + 短信验证）
5. 等待注销审核（1-3天）

> 注销后原AppID `wxb4f6d89904200fd2` 作废，不可恢复。

**第2步：用容蓝实业注册新小程序**

1. 用**容蓝实业**的管理员微信扫码登录 https://mp.weixin.qq.com
2. 点击 **「立即注册」** → 选择 **「小程序」**
3. 填写信息：
   - **主体类型**: 企业（营业执照）
   - **主体名称**: 容蓝实业
   - **营业执照号**: [容蓝实业的信用代码]
   - **管理员姓名/手机/微信**: 容蓝实业的管理员信息
4. 提交后，管理员微信扫码确认
5. 注册成功 → 获得新的 **AppID**（格式：wx开头的一串字符）

> 注册完成后，需要完成微信认证（300元/年），否则无法发布上线。

**第3步：记下新AppID**

注册成功后，在公众平台 → **开发** → **开发管理** → **开发设置** 中查看 AppID。

**重要**：把新AppID复制下来，下一步更新代码需要用到。

## 第二部分：更新代码中的AppID

> 如果走方案A（迁移），AppID不变，此部分可跳过。
> 如果走方案B（重新注册），**必须更新**所有文件中的旧AppID。

### 需要在以下文件中替换AppID

**1. 项目配置文件（2个文件）**

| 文件 | 修改内容 |
|:-----|:---------|
| `liankebao-weapp/project.config.json` | 第5行 `"appid"` 改为新AppID |
| `liankebao-miniapp/project.config.json` | 第37行 `"appid"` 改为新AppID |

**2. 后端代码（3个文件）**

| 文件 | 修改内容 |
|:-----|:---------|
| `backend/app/routers/auth.py:43` | `WECHAT_APPID = "新AppID"` |
| `backend/app/routers/orders.py:39` | 环境变量KEY不变，改默认值 |
| `backend/app/routers/payment.py:803` | 同上 |
| `backend/recharge/routes.py:135` | mock用AppID改为新值 |

**3. 文档文件（建议更新，不影响功能）**

- `README-小程序.md`
- `submission_checklist.md`
- `WECHAT_PAYMENT_PLAN.md`
- `WECHAT_PAY_SETUP_CHECKLIST.md`
- `scripts/miniapp_review_helper.py`

**4. 服务器环境变量（上线前必须配）**

登录服务器，更新环境变量：
```bash
ssh opc@47.116.116.87
sudo vi /etc/systemd/system/chainke.service
# 找到 WEIXIN_APPID= 改为新AppID
# Environment="WEIXIN_APPID=新AppID"
sudo systemctl daemon-reload && sudo systemctl restart chainke
```

## 第三部分：微信公众平台配置

> 这部分**无论方案A还是方案B都要做**。如果是方案A迁移，AppID不变直接配；方案B的话用新AppID配。

### 第1步：登录微信公众平台

1. 打开 https://mp.weixin.qq.com
2. 用管理员微信扫码登录
3. 顶部导航栏点 **「小程序」**（不是公众号）
4. 确认左上角显示你的小程序名称

### 第2步：完成微信认证

> 新注册的小程序必须认证，否则无法发布。迁移后的小程序如果已认证可跳过。

1. 左侧菜单 → **设置** → **基本设置**
2. 找到「微信认证」→ 点击 **「去认证」**
3. 填写容蓝实业的营业执照信息
4. 支付 **300元** 认证费
5. 等待1-2个工作日审核

> ⚠️ 认证费每年交一次，到期不续费小程序无法发布。

### 第3步：配置服务器域名白名单（最重要！）

> **路径**: 左侧菜单 → **开发** → **开发管理** → **顶部Tab「服务器域名」**

**request合法域名**（必须配，不配所有API报404）：

```
https://liankebao.top
```

**操作**：
1. 在输入框输入 `https://liankebao.top`
2. 点击 **「添加」**
3. 点击 **「保存并发布」**

> ⚠️ 配置后5-10分钟生效。开发者工具中可勾选「不校验合法域名」临时绕过。

**uploadFile / downloadFile 合法域名**（如需上传图片）：

```
https://liankebao.top
```

### 第4步：配置业务域名（AI数字名片H5用）

> **路径**: 左侧菜单 → **开发** → **开发管理** → **顶部Tab「业务域名」**

```
https://liankebao.top
```

**操作**：
1. 输入 `https://liankebao.top`
2. 下载微信提供的校验文件 `MP_verify_xxxx.txt`
3. 上传到服务器：
   ```bash
   scp MP_verify_xxxx.txt opc@47.116.116.87:/var/www/html/
   ```
4. 点击 **「保存并发布」**

> ⚠️ 不配置业务域名，小程序内打开web-view（如电子画册）会白屏。

### 第5步：更新用户隐私保护指引

> **路径**: 左侧菜单 → **功能** → **用户隐私保护指引**

1. 点击 **「更新」**
2. 勾选以下信息：
   ```
   ☑ 微信昵称、头像（用户资料）
   ☑ 手机号
   ☑ 位置信息（GEO诊断功能）
   ☑ 订单信息
   ```
3. 用途说明填写：
   ```
   用于用户注册登录、企业家供需匹配、AI数字名片展示、GEO诊断服务和订单管理
   ```
4. 点击 **「提交」**

> ⚠️ 隐私指引未更新 = 审核必被驳回。

### 第6步：选择服务类目

> **路径**: 左侧菜单 → **设置** → **基本设置** → **服务类目**

主类目：**工具 → 企业管理**
副类目：**商业服务 → 企业服务**

需上传容蓝实业的营业执照。

### 第7步：配置小程序信息

> **路径**: 左侧菜单 → **设置** → **基本设置**

| 项目 | 内容 |
|:-----|:------|
| **名称** | 链客宝（需不重名，如被占用可加后缀） |
| **头像** | 使用项目logo（在 `chainke-full/public/` 中） |
| **简介** | AI驱动的企业家供需匹配平台 · 数字名片 · GEO诊断 · 人脉对接 |
| **开放范围** | 全量上线前选「开发版」 |

## 第四部分：开发者工具导入与上传

### 第1步：安装微信开发者工具

下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

### 第2步：构建项目

确保代码最新：
```bash
cd D:\chainke-full\liankebao-weapp
npm install
npm run build:weapp
```

> 如果报错，用 `npx taro build --type weapp 2>&1` 看具体错误。

### 第3步：导入项目

打开开发者工具 → **「导入项目」**

```
项目目录: D:\chainke-full\liankebao-weapp
AppID:    新AppID（方案A: 用原AppID，方案B: 用新AppID）
```

> 注意：项目目录选 `liankebao-weapp` 根目录（不是 dist/）。

### 第4步：开发调试设置

导入后，点击顶部菜单 **详情 → 本地设置**：
```
☑ 不校验合法域名（开发时勾选，上传前取消）
☑ 启用热重载
```

### 第5步：上传代码

1. 工具栏点击 **「上传」**
2. 版本号：`2.0.0`（或递增）
3. 版本描述：
   ```
   链客宝v2.0 — AI数字名片
   功能：AI名片创建编辑、供需匹配引擎、会员体系、GEO诊断
   ```
4. 点击 **「上传」**

### 第6步：提交审核

1. 登录公众平台 → **版本管理**
2. 找到刚上传的开发版本 → 点击 **「提交审核」**
3. 功能页面：勾选首页、AI数字名片、产品池
4. 审核说明：「链客宝AI企业家供需匹配平台，含数字名片创建展示、供需匹配引擎、产品池」
5. 点击 **「提交审核」**

### 第7步：审核通过后发布

审核通过后（1-7天），公众平台 → **版本管理** → 审核通过版本 → 点击 **「发布」** → 全量发布

## 第六部分：常见问题

| # | 问题 | 原因 | 解决 |
|:-:|:-----|:------|:------|
| 1 | 迁移时提示「主体不一致」 | 显臻教育和容蓝实业的营业执照信息对不上 | 确认两家都是正常存续状态，上传清晰照片 |
| 2 | 旧AppID登录不了公众平台 | 显臻教育的认证已过期 | 先续费认证再发起迁移 |
| 3 | 注册新小程序提示「名称被占用」 | 链客宝已被其他主体注册 | 加后缀如「链客宝-企业版」 |
| 4 | 迁移完但API报401 | 服务器环境变量还没更新AppID | SSH登录改 WEIXIN_APPID |
| 5 | API 404 | 域名白名单没配 | 去公众平台配 `liankebao.top` |
| 6 | web-view白屏 | 业务域名没配 | 配 `liankebao.top` + 上传校验文件 |
| 7 | 开发者工具报「AppID不合法」 | project.config.json 的appid没更新 | 改成新AppID |
| 8 | 上传报「代码包过大」 | dist/ 超过2MB | 检查 dist/ 大小，压缩图片 |
