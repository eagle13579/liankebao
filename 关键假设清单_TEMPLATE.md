**假设类型说明**：
- **需求假设**：用户真的需要这个吗？痛点有多痛？
- **方案假设**：我们的方案真能解决痛点吗？用户会用吗？
- **商业假设**：有人愿意付钱吗？能覆盖成本吗？
- **增长假设**：能让更多人来用吗？能快速复制吗？
- **壁垒假设**：别人不能轻易复制吗？护城河在哪？

## 第二步：做减法（三原则筛选TOP 3-5）

### 三原则打分

对每个假设评分（0-10分）：

| # | 假设 | 依赖度 | 不确定性 | 失败成本 | 总分 | 排序 |
|:-:|:-----|:------:|:---------:|:--------:|:----:|:----:|
| | | /10 | /10 | /10 | | |

**总分 = 依赖度 × 不确定性 × 失败成本**

## 第三步：设计验证方案

### 验证原则
- 最小成本：能用调研不做MVP，能用人工不写代码
- 单一变量：每次只验证1个假设
- 前置优先：需求的假设没验证完，不做方案

### 验证矩阵

| 关键假设 | 验证方法 | 成功标准 | 周期 | 成本评估 |
|:---------|:---------|:---------|:----:|:---------|
| 假设1 | 用户访谈/问卷/着陆页/MVP/... | 具体可量化的标准 | ≤2周 | 低/中/高 |
| 假设2 | | | ≤2周 | |

## 第五步：决策

- [ ] **全部关键假设成立 → 进入方案设计**
