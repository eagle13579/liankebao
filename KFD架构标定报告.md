# 链客宝 KFD 架构标定报告

> **版本**: 1.0.0 | **日期**: 2026-06-08 | **方法论**: KFD架构 (Kernel+Feature+DataPack)

## 一、全景总览

链客宝是一个 **企业信任网络+AI数字名片+B2B供需匹配** 平台，服务于中韩企业间信任建立与商机对接。

| 维度 | 数值 |
|:-----|:-----|
| 后端路由模块 | 36个 (FastAPI) |
| 前端页面模块 | 5+个 (React SPA) |
| 前端代码 | 121+ .tsx/.ts 文件 |
| 数据库 | MySQL/SQLite/PostgreSQL 三轨 |
| 部署 | Nginx+Docker+K8s 蓝绿部署 |
| 架构文档 | ARCHITECTURE.md (45KB/924行) |
| 产品文档 | docs/ 下多份架构文档 |
| 安全体系 | 7层纵深防御 |
| AI功能 | 信任评分/学习中心/洞察/增长引擎 |
| 网关 | :5136 统一网关 |

---

## 二、最小内核 (Kernel) 定义

**核心理念**: 链客宝的最小内核 = 用户认证 + 企业名片 + 基础撮合。移除所有Feature后仍能运行的部分。

### 2.1 内核清单

| 组件 | 说明 | 关键文件 |
|:-----|:------|:---------|
| **用户认证** | 登录/注册/Session管理/RBAC | auth.py, 数据库users表 |
| **企业名片** | 名片创建/展示/搜索 | business_card.py, brochure_bridge.py |
| **基础撮合** | 供需发布/匹配推荐/搜索 | matching_events.py, search.py, needs.py, enterprise.py |
| **数据库层** | SQLAlchemy ORM + 连接管理 | database.py, models.py |
| **API网关** | Nginx反向代理+路由分发 | nginx配置, gateway.py |
| **应用入口** | FastAPI应用创建/中间件注册 | main.py |

### 2.2 内核独立性检查

| 检查项 | 状态 |
|:-------|:----:|
| 内核不依赖任何Feature | 🟡 部分依赖 — auth/business_card与AI模块有交叉引用 |
| 内核可独立启动 | 🟢 FastAPI + 数据库即可运行 |
| 内核是稳定的 | 🟢 archives稳定，新增功能在Feature层 |
| 内核提供基础上下文 | 🟢 用户身份/企业身份/基础数据 |

## 九、总结

链客宝已具备良好的KFD迁移基础（体系6.5/10），核心优势在体系维度的完善架构建模和安全体系。最大缺口在：

1. **数据管道系统化**（6/10 → 目标8/10）
2. **创造力维度创新引擎**（5/10 → 目标8/10）
3. **Featur/DataPack注册到记忆宫殿Feature库**（今日可做P0）
4. **测试覆盖与技术债可见性**（6/10 → 目标8/10）

建议先执行P0的四项纯文档级任务，然后选择P1-P2中的方向进行深入。
