## B1: 统一API网关单线程

**根因**: `统一API网关.py` 使用 Python 标准库 `http.server.HTTPServer`，单线程同步模型，一个请求阻塞则全部请求排队。

**修复内容**:
1. 添加 `import socketserver`
2. 新增 `ThreadedHTTPServer` 类，继承 `socketserver.ThreadingMixIn` + `http.server.HTTPServer`
3. 设置 `daemon_threads = True`（守护线程，主进程退出时自动终止）
4. 设置 `allow_reuse_address = True`（端口复用）
5. 启动时改用 `ThreadedHTTPServer(...).serve_forever()`

## B2: SQLite写锁 + debug模式

**根因**:
1. SQLite 默认行为是遇到写锁立即抛错 `database is locked`
2. 启动脚本使用 `--reload`（uvicorn热重载），等效于 Flask debug 模式，单worker+单线程
3. 无 `PRAGMA busy_timeout` 设置，并发写操作直接崩溃

**修复内容**:
1. **database.py**: SQLite引擎添加3个PRAGMA:
   - `PRAGMA busy_timeout=5000` — 写锁时最多等待5秒
   - `PRAGMA journal_mode=WAL` — WAL模式，读写不互相阻塞
   - `PRAGMA synchronous=NORMAL` — 减少fsync频率，提升写入性能
2. **main.py**: 添加 `__main__` 入口，`reload=False` 明确禁用热重载
3. **链客宝全启动器.py**: 移除 `--reload` 参数
4. **本地启动器.py**: 移除 `--reload` 参数

**涉及文件**:

- `D:\链客宝\backend\app\main.py`
- `D:\向海容的知识库\wiki\wiki\记忆宫殿\L5孵化室\产品开发\链客宝\链客宝全启动器.py`

## B3: 无缓存 + 无限流 + 认证mock

**根因**:
1. 静态文件无 Cache-Control 头，每次请求都重新传输
2. 支付接口限流策略宽松，无梯度限流
3. `permission_guard.py` 的 `get_current_user()` 返回硬编码的 `mock_admin` 管理员用户

**修复内容**:
1. **缓存(网关)**: `_serve_static()` 和 H5静态文件服务添加 Cache-Control 头:
   - JS/CSS/字体/图片等不可变资源: `max-age=31536000 (1年) + immutable`
   - HTML/JSON等可变资源: `max-age=3600 (1小时)`
2. **限流**: `rate_limiter.py` ROUTE_LIMITS 增加支付接口梯度限流:
   - `/api/payment/wxpay/unified-order` → 6 req/min（每10秒1次）
   - `/api/payment/wxpay/refund` → 3 req/min（每20秒1次）
   - `/api/v1/payment/` → 10 req/min
3. **认证**: `permission_guard.py` 移除 mock_admin 占位码，替换为真实 JWT 解析:
   - 使用 `python-jose` 解码 JWT token
   - 从 payload 提取 `sub`(用户名)、`user_id`、`role`、`is_superuser`
   - 使用与 `app/auth.py` 一致的 SECRET_KEY 环境变量

**涉及文件**:

- `D:\链客宝\backend\app\rate_limiter.py`
- `D:\向海容的知识库\wiki\wiki\记忆宫殿\L5孵化室\产品开发\链客宝\backend\permission_guard.py`

**验证方法**:
