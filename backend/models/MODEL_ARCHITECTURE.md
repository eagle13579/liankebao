### 2.2 特征工程

#### 用户塔特征 (~50维)
| 类别 | 特征 | 来源 |
|:-----|:------|:------|

#### 企业塔特征 (~80维)
| 类别 | 特征 | 来源 |
|:-----|:------|:------|

| 信用属性 | 企业评级/风险数/异常记录 | 企查查 |

| 特征 | 说明 |
|:-----|:------|

### 2.3 训练设计

```
训练数据:
  - 正样本: 成功对接/互相好评的对(标签=1)
  - 负样本: 被拒绝/无响应对(标签=0)
  - 负采样: 随机配对(比例1:5)

损失函数:
  L = BCE(y_true, y_pred) + λ1 * L_pairwise + λ2 * L_contrastive
  
  BCE: 二分类交叉熵
  Pairwise Loss: 让正样本对分数>负样本对 (margin=0.3)
  Contrastive Loss: 拉近相似用户/企业嵌入距离

评估指标:
  - AUC (主指标, 目标>0.85)
  - Precision@K (K=5/10/20)
  - Recall@K
  - NDCG@K
  - 多样性 (MMR后的结果多样性)

训练配置:
  - Batch Size: 256
  - Learning Rate: 1e-3 → 1e-5 (余弦退火)
  - Optimizer: AdamW
  - Epochs: 50 (早停 patience=5)
  - Validation Split: 80/20
```

### 3.1 双塔架构

```
┌────────────────────┐    ┌────────────────────┐
│   Query塔           │    │   Candidate塔        │
│                     │    │                     │
│  用户嵌入 (128d)    │    │  企业嵌入 (128d)     │
│  + 场景嵌入 (32d)   │    │  + 行业嵌入 (32d)    │
│         ↓           │    │         ↓            │
│  DNN: 128→256→128   │    │  DNN: 128→256→128   │
│         ↓           │    │         ↓            │
│  L2 Normalize       │    │  L2 Normalize       │
│    Query向量 (128d)  │    │  Candidate向量 (128d)│
└────────┬───────────┘    └───────────┬──────────┘
         │                            │
         └──────────┬────────────────┘
                    │
             内积 → sim(q, c)
                    │
              MMR多样性重排序
                    │
              Top-N 推荐结果
```

### 3.2 冷启动策略 (M4)

| 场景 | 策略 | 模型 |
|:-----|:------|:------|
| 纯新用户(未注册) | 行业热门推荐 | 规则 |

## 四、M3 信任评分模型 — GBDT+规则融合

### 4.1 双级评分架构

```
第一级: 规则引擎 (应急·可解释)
  企业基础评分 = 注册资本分 + 存续年限分 + 行业风险分
  不依赖训练数据, 实时可解释

第二级: GBDT模型 (精准·数据驱动)
  输入: 企业特征 + 行为特征 + 图谱特征
  输出: 0~100 信任分
  算法: LightGBM / XGBoost

融合: 
  最终信任分 = α * 规则分 + (1-α) * GBDT分
  α = 1 / (1 + 训练样本数/1000)  — 数据越多越依赖模型
```

### 4.2 特征体系

| 类别 | 特征 | 数量 |
|:-----|:------|:----:|
| 工商特征 | 注册资本/年限/法人资质/股东稳定性 | 15 |
| 信用特征 | 风险评级/异常记录数/处罚次数 | 10 |
| 行为特征 | 平台活跃度/匹配响应率/成交履约率 | 10 |
| 图谱特征 | 合作伙伴平均信任分/行业信誉 | 8 |
| 外部特征 | 天眼查风险分/企查查信用分 | 5 |

### 7.3 Bandit探索-利用策略

```python

def select_action(user, candidates, epsilon=0.1):
    if random.random() < epsilon:
        return random.choice(candidates)  # 探索
    else:
        return model.predict(user, candidates)  # 利用
    
# epsilon 随用户活跃度衰减
epsilon = max(0.01, 0.3 * (0.9 ** user.match_count))
```

## 十、设计原则

1. **模型即Feature**: 每个模型封装为独立Feature, 有标准接口(输入schema/输出schema/元数据), 可排列组合
2. **从规则到模型渐进**: P0规则→P1浅层模型→P2深度模型, 每阶段可独立验证
3. **数据先行**: 没有训练数据不训练模型, 先采集再建模
4. **回退安全**: 每个模型都有降级策略(模型不可用→规则→默认值), 永不崩溃
5. **可解释性**: 匹配分数可分解为各塔贡献度, 信任评分可溯源
