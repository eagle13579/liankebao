## 二、AI 功能分类

### 2.1 名片AI（Card AI）
- OCR扫描 → 字段提取（姓名/职位/公司/手机/邮箱/微信/地址/官网）
- 数字名片生成 + 翻页图册展示
- 名片 → 供需匹配一键触发
- **当前管线**: `scan_card() → extract_fields() → generate_digital_card() → match_supply_demand()`

### 2.2 AI供需匹配（Matching Engine）
- 类目匹配（category相等 → 30分）
- 价格区间匹配（预算重叠 → 30分）
- **TF-IDF语义匹配**（文本相似度 → 40分）— v2核心升级
- **向量增强**（关闭中，启用后BM25+embedding混合重排序）
- A/B测试框架（`?strategy=v1|v2`）

### 2.3 个性化推荐（Recommend）
- 有行为用户：同类产品推荐（category+tags相似）
- 无行为用户：热门推荐（按浏览量）
- 搜索关键词兜底推荐

### 2.4 搜索系统（Search）
- 中文分词搜索（jieba）
- FTS5全文索引（SQLite）
- 多维度筛选（分类/地区/价格区间）
- **向量重排序**（关闭中，启用后提升语义相关性）

### 2.5 信任评估（基础）
- `EnterpriseRelation` 模型（投资/竞争/供应/子公司/合作伙伴/客户关系）
- 企业公域信息采集（`enterprise_crawler.py`）
- **缺少**：AI驱动的信用评分、欺诈检测、交易风险评估

## 三、用户旅程中的 AI 接触点

```
注册 → 发名片 → 搜索/浏览 → 匹配 → 询盘沟通 → 成交 → 信任沉淀
│       │          │           │        │          │        │
│       │          │           │        │          │        └─ AI信任分
│       │          │           │        │          └─ AI成交预测
│       │          │           │        └─ AI智能回复/需求澄清
│       │          │           └─ AI供需匹配 + 匹配分数/原因
│       │          └─ AI搜索（语义搜索+向量重排序）
│       └─ AI名片扫描 → 自动提取字段 → 匹配触发
└─ AI推荐引导完成profile + 痛点选择
```

### 各接触点当前AI覆盖情况

| 接触点 | 当前AI | 缺口 |
|--------|--------|------|
| **① 注册** | 痛点采集→影响推荐 | 无AI引导填写、无智能profile补全 |
| **② 发名片** | OCR扫描+字段提取 ✅ | DeepSeek API未接入，OCR精度受限 |
| **③ 搜索/浏览** | FTS5+jieba ✅ | 向量重排序关闭，无对话式搜索 |
| **④ 匹配** | TF-IDF语义+类目+价格 ✅ | 向量增强关闭，无实时对话匹配 |
| **⑤ 询盘沟通** | ❌ | 无AI客服、无智能沟通建议 |
| **⑥ 成交** | ❌ | 无AI成交预测、无智能定价建议 |
| **⑦ 信任沉淀** | 企业关系模型(空) ❌ | 无AI信用评分、无交易风险提示 |

## 四、产品功能重构建议

### 4.1 AI名片 → "智能名片3.0"

**现状**: 上传名片→OCR提取→手动确认→生成数字名片

**重构方案**:
```
前端入口           AI后台                      用户交互
┌─────────┐    ┌──────────────┐    ┌─────────────────────┐
│拍照/上传 │ →  │ DeepSeek OCR │ →  │ 字段预览 + 一键确认  │
│微信导入  │    │ + 字段提取    │    │ (可编辑纠错)         │
│通讯录导入│    │ + 企业信息补全│    │ + "一键匹配供需"按钮  │
└─────────┘    └──────────────┘    └─────────────────────┘
```

**前端表现**:
- 微信小程序/PC端：拍照即扫，3秒出结果
- 名片详情页：翻页图册式展示（已有brochure_bridge）
- 名片列表页：AI标签（"近期活跃""高匹配度"）
- 分享名片：一键转发微信，对方直接查看数字名片+供需信息

### 4.2 AI供需匹配 → "智能商机推荐"

**现状**: 被动触发（用户点匹配才跑）

**重构方案**:
| 模式 | 触发时机 | 前端表现 |
|------|---------|---------|
| **主动推送** | 新需求/新产品发布时 | 微信模板消息："发现3个匹配您的商机" |
| **首页Feed** | 每次打开App | "为您推荐"卡片流 + 匹配分数 + 匹配原因 |
| **名片联动** | 扫描他人名片时 | "您和TA可能有这些合作机会" |
| **搜索结果** | 搜索关键词时 | 搜索结果顶部插入匹配卡片 |

**前端元素**: 匹配分数进度条(0-100分) + 匹配理由标签("类目匹配""关键词匹配""预算匹配") + "立即沟通"按钮

### 4.3 AI推荐 → "个性化为先"

**现状**: 简单规则（同类/热门）

**升级路径**:
1. P0: 开启向量搜索（配置即可用）
2. P1: 融合注册痛点标签进推荐权重
3. P2: 冷启动→依据行业+区域+企业规模做协同过滤
4. P3: LLM生成推荐理由（"这个供应商和您在同一个园区"）

### 4.4 AI搜索 → "语义搜索"

**现状**: 关键词匹配，无语义理解

**重构**:
- 启用向量重排序（`RERANK_WEIGHT=0.3` 即可）
- 搜索框加AI入口："描述您需要的产品..."（自然语言输入）
- 搜索结果标记AI匹配度

### 4.5 信任评估 → "AI信任分"（新功能）

需要新建，当前完全缺失

| 维度 | 数据来源 | AI处理 |
|------|---------|--------|
| **企业验证** | 企业工商信息(crawler) + 关系图谱 | AI验证真实性 → 基础信任分 |
| **交易记录** | 订单完成率+纠纷率 | 机器学习 → 交易信誉分 |
| **活跃度** | 登录频次+响应速度 | 行为评分 → 活跃度标记 |
| **关系网络** | 联系人数量+合作企业 | 图算法 → 社交资本分 |

**前端展示**: 企业主页展示"AI信任分⭐⭐⭐⭐"(1-5分)，类似闲鱼信用分

## 五、竞品参考与差异化

### 5.1 AI名片类 — 名片全能王(CamCard)

| 维度 | 名片全能王 | 链客宝AI差异化方向 |
|------|-----------|----------------|
| OCR精度 | 百度OCR+自研模型，98%+ | **接入DeepSeek Vision + 企业信息补全**，走轻量化路线 |
| 名片管理 | 强（通讯录+分组+备注） | **弱（当前仅存储）→ 改造为CRM入口** |
| 社交传播 | 弱（仅分享图片） | **强（翻页图册+供需信息+一键匹配）** |
| 商业闭环 | 无 | **强：名片→匹配→沟通→成交完整链路** |

**链客宝AI核心差异**: 名片不是终点，是商业匹配的起点。

### 5.2 AI匹配平台 — 生意汇/脉脉/B2B平台

| 维度 | 传统B2B/脉脉 | 链客宝AI差异化 |
|------|-------------|------------|
| 匹配精度 | 关键词搜索+类目筛选 | **TF-IDF语义+向量重排序+多维度加权** |
| 信任体系 | 依赖平台背书 | **AI信任分+企业关系图谱** |
| 轻量化 | 重平台运营 | **微信生态轻量入局，名片即入口** |
| AI智能化 | 搜索推荐独立 | **名片→匹配→推荐→搜索四合一AI** |

### 5.3 链客宝AI差异化定位

```
"AI驱动的商业连接器 — 名片社交+智能匹配"
```

三大差异化支柱：
1. **名片即入口**: 不同于传统B2B需要手动发布供需，扫名片就触发匹配
2. **全链路AI**: OCR→匹配→推荐→搜索→信任，端到端AI覆盖
3. **微信原生**: 小程序+微信分享+模板消息，零下载门槛
