# 链客宝AI服务等级协议 (SLA)

> **文档版本**: v1.0
> **生效日期**: 2026-05-29
> **适用范围**: 链客宝AI生产环境
> **负责人**: 链客宝AI运维团队

---

## 1. 服务可用性承诺

| 指标 | 承诺值 | 计算方式 |
|------|--------|----------|
| 月度可用性 | **≥ 99.5%** | (总分钟数 - 不可用分钟数) / 总分钟数 × 100% |
| 年度可用性 | **≥ 99.0%** | 12个月可用性平均值 |

**可用性计算公式**:

```
可用性(%) = (T - D) / T × 100%

T = 月度总分钟数
D = 月度不可用分钟数（排除计划内维护窗口）
```

**不可用定义**: 用户无法通过正常方式访问链客宝AIAPI或前端页面，且该故障并非由用户端原因、第三方不可抗力或计划内维护造成。

---

## 2. API 响应时间承诺

| 百分位 | 承诺值 | 测量方式 |
|--------|--------|----------|
| P50 (中位数) | **< 200ms** | 服务端请求处理时间 |
| P95 (第95百分位) | **< 500ms** | 服务端请求处理时间 |
| P99 (第99百分位) | **< 2000ms** | 服务端请求处理时间 |

**排除项**:
- 大文件上传/下载请求
- 批量数据导出请求
- 首次冷启动请求
- 因客户端网络延迟导致的耗时

---

## 3. 故障响应与恢复时间

### 3.1 故障等级定义

| 等级 | 标签 | 定义 | 示例 |
|------|------|------|------|
| **P0** | 紧急故障 | 核心业务完全不可用，影响所有用户 | 服务宕机、数据库崩溃、支付完全中断 |
| **P1** | 严重故障 | 主要功能受损，影响大量用户 | 登录失败率>20%、API响应超时率>30% |
| **P2** | 一般故障 | 非核心功能受损，影响部分用户 | 单个功能异常、管理后台不可用 |
| **P3** | 轻微故障 | 功能可用但有轻微影响 | UI显示问题、非关键告警 |

### 3.2 响应与恢复 SLA

| 等级 | 响应时间 | 恢复时间 (RTO) | 通知对象 |
|------|----------|----------------|----------|
| **P0** | ≤ 15 分钟 | **≤ 1 小时** | 运维团队 + 技术负责人 |
| **P1** | ≤ 30 分钟 | **≤ 4 小时** | 运维团队 |
| **P2** | ≤ 2 小时 | **≤ 24 小时** | 运维团队（工作时间） |
| **P3** | ≤ 8 小时 | **≤ 72 小时** | 记录跟踪 |

**恢复时间目标 (RTO)**:
- P0: 1小时内恢复服务
- P1: 4小时内恢复服务
- P2: 24小时内恢复服务

**数据恢复点目标 (RPO)**:
- 数据库: ≤ 5 分钟（自动备份）
- 文件存储: ≤ 24 小时（每日快照）

## 4. 维护窗口

### 4.1 计划内维护

| 项目 | 说明 |
|------|------|
| 频率 | **每季度一次**（3/6/9/12月） |
| 通知提前期 | **至少提前 7 天** 通过以下渠道通知： |
| 通知渠道 | 系统公告、注册邮箱、钉钉群 |
| 预计时长 | ≤ 4 小时 |
| 窗口时间 | 北京时间 02:00 - 06:00（业务低峰期） |

### 4.2 紧急维护

在出现安全漏洞或严重Bug需要立即修复时，可在**提前2小时**通知后进行紧急维护。

### 4.3 维护排除

计划内维护时间不计入服务不可用时间，不影响可用性计算。

## 5. 服务赔偿条款

### 5.1 赔偿条件

当月度可用性低于承诺值（99.5%）时，用户可申请服务赔偿。

### 5.2 赔偿标准

| 月度可用性 | 赔偿额度 |
|------------|----------|
| 99.0% ~ 99.4% | 当月服务费的 **10%** |
| 98.0% ~ 98.9% | 当月服务费的 **25%** |
| 95.0% ~ 97.9% | 当月服务费的 **50%** |
| < 95.0% | 当月服务费的 **100%** |

### 5.3 赔偿申请流程

1. 用户通过工单系统提交赔偿申请
2. 提供故障时间、影响范围说明
3. 链客宝AI运维团队在 5 个工作日内核实
4. 核实通过后，赔偿金额在下期账单中抵扣

### 5.4 赔偿上限

- 单次事件赔偿上限: 当月服务费的 100%
- 年度累计赔偿上限: 3个月服务费

### 5.5 免责条款

以下情况不计入不可用时间，不触发赔偿：

- 计划内维护
- 第三方服务故障（云服务商、CDN、DNS、支付网关等）
- 不可抗力（自然灾害、战争、疫情、政府行为等）
- 用户自身原因导致的故障（错误配置、恶意攻击等）
- 非生产环境（测试、预发布、开发环境）
- 因API限流导致的请求失败

## 7. 安全承诺

- 所有API通信使用 TLS 1.2+ 加密
- 用户密码使用 bcrypt 加密存储
- 支付信息使用 AES-256-GCM 加密
- 数据定期备份（数据库每分钟 + 每日完整备份）
- 每年至少进行一次安全审计
