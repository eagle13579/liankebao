## 内核门禁

### 门禁1：这个功能强化了内核吗？

| 角色 | 判断 | 评分(1-10) |
|:-----|:----:|:----------:|
| 创始人 | ✅/❌ | /10 |
| 产品经理 | ✅/❌ | /10 |
| 程序员 | ✅/❌ | /10 |

**三人都打≥7分 → 通过进入排期**
**任何一人打<5分 → 退回重新设计**

### 门禁2：如果只能做一个功能，还选这个吗？

```
创始人的答案：
产品经理的答案：
程序员的答案：
```

**三人答案一致 → 通过**
**答案不一致 → 回到核心任务句式重新讨论**

### 门禁3：这个功能不做，用户会抱怨吗？

```
如果不上这个功能，用户会说："我想要的功能怎么没有？"
如果答案模糊或犹豫 → 这个功能不是内核直接相关
```

---

## 决策

- [ ] 通过三个门禁 → 进入需求评审
- [ ] 有一个门禁不通过 → 重新设计或放弃

- [ ] 会轻微影响 → P2/P3
